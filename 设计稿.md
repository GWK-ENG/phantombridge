# 🌌 PhantomBridge - AI 浏览器底层控制网关 总体设计稿

## 一、 项目定位与核心目标

本项目并非一个面向普通人类用户的“聊天插件”，而是一个**专为 AI Agent 和自动化 CLI 打造的浏览器运行时环境**。
它将极其复杂的浏览器 DOM 结构、反爬机制、多标签页状态管理，封装为大模型能够轻松调用的原子级 API（全面兼容 Model Context Protocol, MCP）。

## 二、 全局架构设计：四层分离模型

系统从上到下严格划分为四层，确保高并发下的稳定性与指令的精准触达：

### 1. 业务大脑层 (The Brain - AI Agents)

* **接入方**：OpenClaw, 自定义 Python/Node.js 脚本, 任何支持 MCP 的客户端。
* **职责**：大模型推理、任务规划（如“登录账号”、“抓取订单”）。
* **输出**：向网关层发送标准化的 JSON-RPC 或 MCP 指令。

### 2. 本地路由网关 (The Gateway - Node.js 守护进程)

* **运行环境**：Windows 后台常驻服务。
* **核心职责**：
* **进程调度**：接收到新任务时，静默拉起 `chrome.exe`，并分配独立的 `--profile-directory`（实现环境隔离与多开）。
* **协议中转**：将外部的 HTTP/WebSocket/MCP 请求，打包成符合 Chrome Native Messaging 标准的字节流（携带数据长度前缀）。
* **连接池管理**：维护一个映射表 `Map<TaskID, BrowserInstance>`，确保多 Agent 并发时不串线。



### 3. 浏览器中枢控制层 (The Commander - 扩展 Service Worker)

* **核心职责**：
* **长连接保持**：通过 `chrome.runtime.connectNative` 与本地网关保持心跳连接。
* **内核级动作注入**：独占式调用 `chrome.debugger` (CDP)，执行绕过前端防线的“真实鼠标移动、按下、释放”和“原生键盘敲击”。
* **标签页管理**：创建隐藏标签、切换、关闭。



### 4. 页面感知层 (The Sensors - Content Scripts 探针)

* **核心职责**：
* **DOM 清洗器 (Readability)**：剥离无用 CSS/JS，提取极简的 Markdown 供大模型阅读。
* **坐标系映射 (Spatial Mapping)**：遍历页面交互元素，打上唯一 ID 标签（如 `[btn-8]`），并利用 `getBoundingClientRect()` 计算出绝对屏幕坐标，上报给中枢层。
* **后台保活欺骗**：覆写浏览器的页面可见性 API，防止切换标签时页面冻结。

## 三、 核心工作流设计

### 工作流 A：零干预“幽灵启动” (Phantom Startup)

为了实现 AI 完全接管，不需要人类点击 Chrome：

1. Agent 下发指令：`{"action": "start_session", "task_id": "job_123"}`
2. Node.js 网关在 Windows 底层执行命令行：
`"C:\Program Files\Google\Chrome\Application\chrome.exe" --profile-directory="Profile_Job_123" --load-extension="C:\path\to\extension" --new-window "https://target.com"`
3. Chrome 启动后，扩展内部立即调用 Native Messaging 连回网关，完成握手。

### 工作流 B：空间映射与真实点击 (Spatial Mapping & CDP Click)

解决“人机验证被拦截”及“坐标精准度”问题：

1. Agent 请求：`{"action": "get_interactive_elements"}`。
2. 探针脚本 (Content Script) 在页面上注入不可见的标记，返回类似这样的数据映射：
`{"id": "input-1", "type": "username", "x": 450, "y": 300, "width": 200, "height": 40}`
3. Agent 决策后下发指令：`{"action": "real_click", "target_x": 450, "target_y": 300}`。
4. Service Worker 调用 `chrome.debugger`，精准在 `(450, 300)` 触发底层电信号级别的 `mousePressed` 和 `mouseReleased`。哪怕人类此时正在操作其他软件，只要不把鼠标强行点进这个指定的 Chrome 窗口，就不会产生冲突。

### 工作流 C：后台静默保活 (Visibility Override)

解决人类切换窗口或 Agent 在后台多开标签时，React/Vue 页面自动暂停渲染的问题。
探针脚本 (Content Script) 在页面最早期注入以下代码，强行篡改状态：

```javascript
// 欺骗前端框架，让它永远以为自己处于前台激活状态
Object.defineProperty(document, 'visibilityState', {
    get: function () { return 'visible'; }
});
Object.defineProperty(document, 'hidden', {
    get: function () { return false; }
});
// 屏蔽 visibilitychange 事件
document.addEventListener('visibilitychange', function(e) {
    e.stopImmediatePropagation();
}, true);

```

## 四、 接口协议设计 (MCP 兼容草案)

为了无缝对接你提到的各种 CLI 和 Docker MCP 生态，本地网关对外暴露的 Tool 接口标准如下：

* **`browser_navigate(url)`**: 打开或跳转指定网页。
* **`browser_get_markdown()`**: 获取当前页面的高纯度 Markdown（包含元素 ID 映射）。
* **`browser_cdp_click(x, y)`**: 发送内核级点击指令。
* **`browser_cdp_type(text)`**: 发送内核级键盘输入指令。
* **`browser_evaluate_js(code)`**: 在当前页面上下文中注入并执行自定义 JavaScript 脚本。

## 五、 项目演进与高价值路线

这个底层基建一旦跑通，它不仅仅是一个玩具，而是一个具备极高护城河的 **RPA (机器人流程自动化) 核心引擎**。

1. **第一阶段 (MVP)**：跑通 Node.js 网关 <-> Chrome 扩展的 Native Messaging 通信，实现一个网页的 Markdown 抓取和一次 CDP 点击。
2. **第二阶段 (协议化)**：将其完整封装为一个标准的 MCP Server。这意味着无需额外开发，市面上所有支持 MCP 的前沿 AI 工具（如 Claude Desktop）都能瞬间拥有控制你电脑浏览器的能力。
3. **第三阶段 (高净值转化)**：这种绕过反爬机制、支持多开环境隔离的浏览器控制方案，在数据采集、自动化电商运营、社交媒体自动化管理等领域是绝对的刚需。你可以将其打包为企业级 API 接口或付费授权的开发者工具箱。
